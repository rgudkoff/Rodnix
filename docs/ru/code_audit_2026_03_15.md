## Остаточные проблемы

- В репозитории остаются identifier-level упоминания сторонних систем в именах файлов, каталогов, типов, функций и make-целях:
  - `kernel/linux/*`
  - `drivers/fabric/net/e1000_freebsd_compat.c`
  - `kernel/net/bsd_*`
  - `sync-bsd-abi`, `check_bsd_abi_headers.py`, `sync_bsd_abi_headers.py`
- В репозитории остаются документационные и служебные файлы на английском языке вне этого отчёта и каталога `docs/ru/`.
- В каталоге `third_party/` находятся внешние импортированные материалы с исходными именами и заголовками; они не редактировались.
- В рабочем дереве уже были несвязанные пользовательские изменения вне области этого аудита; они не затрагивались.

## Рекомендации для следующего этапа

- Выполнить отдельный безопасный рефакторинг имён compatibility-слоя:
  - `kernel/linux` -> проектно-нейтральное имя ABI-слоя;
  - `kernel/net/bsd_*` -> проектно-нейтральные сетевые совместимые интерфейсы;
  - `e1000_freebsd_compat.c` -> нейтральное имя shim-модуля.
- Перевести оставшуюся документацию верхнего уровня (`*.md`) на русский язык отдельной серией правок, чтобы не смешивать локализацию с кодовым аудитом.
- Для generated headers сначала обновить генераторы, затем регенерировать файлы, чтобы исключить повторное появление старых комментариев.
