# План импорта BSD-компонентов

Этот документ фиксирует практичный порядок заимствования кода из
оригинальных BSD-репозиториев (в первую очередь FreeBSD), чтобы
ускорить разработку ядра RodNIX без потери управляемости.

## Принципы

- Источник правды: оригинальные BSD-деревья (`src/sys/...`), а не
  фрагменты из сторонних форков.
- Импортируем поэтапно: маленькие, проверяемые порции с тестом/смоком.
- Сначала инфраструктура ядра (контейнеры/примитивы), затем политики.
- Каждый импорт должен иметь запись в `third_party/bsd/SOURCES.md`.

## Что уже внедрено

- `include/bsd/sys/queue.h` (минимальный `TAILQ`-поднабор).
- `include/bsd/sys/tree.h` (минимальные RB-совместимые примитивы).
- Scheduler runqueue переведен на `TAILQ`.
- Поиск `task` по `task_id` переведен с линейного обхода на индекс-дерево.
- Заголовки `include/bsd/sys/*` уже используются в runtime-коде ядра:
  - `kernel/common/waitq.h`
  - `kernel/common/scheduler/internal.h`
- `include/bsd/sys/*` считается рабочим compatibility-layer:
  любые изменения должны проходить через процесс импорта и фиксироваться в
  `third_party/bsd/SOURCES.md`.

## Приоритет P0 (делать в первую очередь)

1. Полноценный `sys/queue.h`:
   привести макросы к более полному BSD-подмножеству (`LIST`, `TAILQ`,
   безопасные вариации обхода, явные инварианты удаления/вставки).
2. Полноценный `sys/tree.h`:
   заменить текущий unbalanced-BST путь на штатные RB-макросы BSD.
3. Очереди ожидания ядра:
   унифицировать wait/sleep очереди через `TAILQ` с единым API
   (`waitq_init`, `waitq_enqueue`, `waitq_wake_one`, `waitq_wake_all`).

## Приоритет P1 (после стабилизации P0)

1. `sys/bitset.h` и связанные bit-утилиты:
   для cpuset/affinity и масок событий.
2. `sys/param.h`, `sys/systm.h` (минимально):
   только те макросы/вспомогательные функции, которые реально нужны ядру.
3. Выборочно `sys/queue.h`-cleanup по подсистемам:
   IPC очереди, VFS списки, сетевые очереди.

## Приоритет P2 (когда база готова)

1. Элементы callout/timer-подхода BSD для унификации таймерных событий.
2. Фрагменты `sys/eventhandler.h`-подобной модели подписок ядра.
3. Фрагменты lock/witness-подхода (без прямого копирования сложной обвязки)
   как ориентир для диагностики инверсий приоритета.

## Что не импортируем напрямую

- Userland (`sh`, `zsh`, libc) в ядро.
- Большие подсистемы целиком без адаптационного слоя.
- Код без понятной лицензии/источника или без записи provenance.

## Definition of Done для каждого импорта

1. Добавлен источник и лицензия в `third_party/bsd/SOURCES.md` и
   `THIRD_PARTY_NOTICES.md`.
2. Есть минимальные тесты/смок (`make` + `./scripts/ci/smoke_qemu.sh`).
3. Нет регрессий в boot path и shell/console.
4. Документация обновлена (минимум один профильный раздел в `docs/ru`).
