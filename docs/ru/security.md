# Учёт и права

## Цели

- Реальные/эффективные UID/GID.
- Базовая модель прав доступа.
- Дальнейшее расширение через MAC‑крючки.

## MVP

- Реальные UID/GID и эффективные UID/GID.
- Минимальный набор проверок на уровне syscalls.

## Текущая реализация (v0)

- Идентификаторы UID/GID добавлены в `task_t` (реальные и эффективные).
- Значения по умолчанию: `uid=0`, `gid=0`, `euid=0`, `egid=0`.
- Проверки прав ещё не реализованы.

## Планы

- Расширение модели прав.
- MAC‑крючки как расширяемый механизм.

## Инварианты

- Проверки прав выполняются до выполнения операции.
- Учёт прав не должен быть распределен по драйверам.

## Где смотреть в коде

- `kernel/core/task.h`, `kernel/common/task.c`.
