# Прерывания и таймеры

## Цели

- Предсказуемая обработка ISR/IRQ.
- Минимизация ручных хаков в EOI/маскировании.
- Базовый таймер для будущего планировщика.

## Что уже есть

- Инициализация GDT и IDT.
- ISR и IRQ обработчики.
- PIC remap.
- PIT таймер.
- Подготовленный путь IRQ для клавиатуры.
- IRQ32 используется как единый вход для таймерного тика и старта первого потока.

## Что планируется

- Единая точка маршрутизации IRQ через Fabric.
- Устойчивый таймерный тик как источник планирования.
- Нормализованный EOI путь без дублирования.
- Устранить двойной `scheduler_tick()` на IRQ32 (один тик на прерывание).

## Инварианты

- Любой IRQ должен иметь однозначный путь обработки.
- Обработчики должны быть максимально короткими.
- Любые печати и отладка не должны ломать IRQ путь.

## Контракты

- Регистрация обработчиков должна быть явной и проверяемой.
- Код архитектуры отвечает за low-level вход/выход из прерываний.

## Где смотреть в коде

- `kernel/interrupts` и `kernel/arch/x86_64`.
