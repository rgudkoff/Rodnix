# Различия: документация vs реализация (на текущий момент)

Документ фиксирует расхождения между описанной архитектурой и фактической
реализацией в коде. Цель — дать точный список "что уже есть" и "чего еще
нет", чтобы планирование и разработка были синхронизированы.

## Планировщик

Документация:
- Требуется приоритетный планировщик + таймслайс (MLQ/priority queues).

Реализация:
- `kernel/common/scheduler.c` — простой FIFO ready-queue, фактически RR.
- Приоритеты есть в API, но очереди по приоритетам отсутствуют.
- Политики CFS/FIFO/PRIORITY указаны в enum, но не реализованы.

Разница:
- Нет MLQ/priority queues и политики приоритетов.
- Таймслайс есть, но нет перераспределения по приоритетам.

## Таймерный тик (IRQ32)

Документация:
- Таймерный тик — единый источник преэмпции (один tick на IRQ).

Реализация:
- Для LAPIC: `apic_timer_handler()` уже вызывает `scheduler_tick()`.
- Дополнительно `interrupt_dispatch()` при `vector == 32` вызывает
  `scheduler_tick()` и `scheduler_switch_from_irq()`.

Разница:
- Тик может учитываться дважды на один IRQ32 (двойной `scheduler_tick()`),
  что расходится с ожидаемой моделью "один тик на прерывание".

## IPC и порты

Документация:
- Сообщения фикс/перем длины.
- Передача портов в сообщениях.
- Граница слоев через IDL + автогенерацию.

Реализация:
- `kernel/common/ipc.c` — заготовка, очереди/таблицы портов не реализованы.
- `ipc_message_t` — фиксированный буфер 4096 байт.
- Нет механизма передачи портов и проверки прав.

Разница:
- Нет переменной длины сообщений.
- Нет передачи портов.
- Нет инфраструктуры очередей/таблиц портов.
- Нет IDL/генерации.

## Userland и bootstrap

Документация:
- Bootstrap‑порт и bootstrap‑сервер в userland (раздача портов сервисов).

Реализация:
- Userland отсутствует, bootstrap‑сервер не реализован.

Разница:
- Нет userland процесса.
- Нет bootstrap‑порта и протокола раздачи портов.

## Системные вызовы (BSD‑слой)

Документация:
- Таблица syscalls в BSD‑слое.
- Вход через traps из userland.

Реализация:
- BSD‑слоя и таблицы syscalls нет.
- Нет trap‑входа из userland.

Разница:
- Полная подсистема syscalls отсутствует.

## VFS

Документация:
- Модель vnode/inode.
- Name cache.
- Монтирование.
- Initrd/ramfs для старта.

Реализация:
- `kernel/fs/vfs.c` — минимальный RAMFS с деревом узлов.
- Нет vnode/inode модели.
- Нет name cache.
- Нет монтирования.
- Initrd не реализован.

Разница:
- VFS пока представляет собой единый RAMFS без vnode/inode и mount.
- Нет name cache и initrd.

## Сеть

Документация:
- Позже: BSD‑сокеты (AF_INET), порядок loopback → UDP → TCP.

Реализация:
- Сетевой стек отсутствует.

Разница:
- Полная подсистема сети отсутствует.

## Учёт и права

Документация:
- Реальные/эффективные UID/GID.
- Базовая модель прав, позже MAC‑крючки.

Реализация:
- Механизмов UID/GID и прав нет.

Разница:
- Подсистема учёта/прав отсутствует.

## IDL и генерация

Документация:
- `.defs` + генератор клиент/серверных стабов и диспетчеров.

Реализация:
- IDL и генератор отсутствуют.

Разница:
- Нет инфраструктуры для автогенерации межслойных интерфейсов.
