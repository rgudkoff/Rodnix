# ABI (userland ↔ kernel)

## Цели

- фиксированная граница между userland и ядром;
- стабильность вызовов и структур;
- предсказуемая совместимость между версиями.

## Версия ABI

- глобальная версия ABI: `RDNX_ABI_VERSION` (`include/abi.h`);
- любое изменение соглашений вызова или layout структур требует поднятия версии ABI.

## Соглашение вызовов (x86_64)

- вход через `int 0x80` (`SYSCALL_VECTOR`);
- `rax` — номер syscall;
- `rdi, rsi, rdx, r10, r8, r9` — аргументы 1..6;
- `rax` — код возврата.

## Версионирование структур

Любая структура, пересекающая границы подсистем или userland, должна иметь заголовок ABI:

```
rdnx_abi_header_t hdr;
```

Правила:

- `hdr.abi_version` должен быть равен `RDNX_ABI_VERSION`;
- `hdr.size` фиксирует размер структуры для совместимости;
- обработчики обязаны проверять `abi_version` и `size`.

Для таблиц операций (`*_ops_v1`) действует то же правило: ops‑структура должна содержать ABI‑header, а потребитель обязан проверять `abi_version/size` перед использованием.

## Текущие ABI‑структуры

- `utsname_t`
- `fabric_service_t`
- `keyboard_ops_t`
- `ipc_message_t`

## Стабильность интерфейсов

- syscall‑номера фиксируются и не меняются;
- новые syscalls добавляются в конец таблицы;
- удаление syscall возможно только с пометкой `RESERVED`.

## Совместимость

- minor‑изменения допускаются только при сохранении ABI;
- если ABI изменён, требуется новая major‑версия ядра.
