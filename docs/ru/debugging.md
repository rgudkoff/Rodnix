# Диагностика и отладка

## Цели

- Предсказуемая диагностика без влияния на критические пути.
- Возможность быстро локализовать регрессии.

## Базовые инструменты

- QEMU для запуска.
- GDB для отладки ядра.
- Serial логирование как основной канал.

## Рекомендации

- На раннем этапе используем минимум логирования в IRQ путях.
- Любые эксперименты с выводом фиксируем в отдельной ветке.
- Сохраняем логи запуска в `boot.log`, если воспроизводим проблему.

## Минимальный сценарий отладки

```bash
make debug
qemu-system-x86_64 -m 64M -kernel build/rodnix.kernel -s -S &
gdb build/rodnix.kernel
```
