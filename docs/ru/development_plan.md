# План развития (кратко)

Этот файл — сжатое резюме. Полный рабочий план: `docs/PLAN.md`.

## Очередность (факт → документация)

0. Базовые предпосылки
   - IDL/генератор: `.defs` → стабы/диспетчеры.
   - IPC (MVP): порты, очереди, fixed‑size сообщения.
   - IPC (расширение): variable‑size + передача портов.

1. Переход в userland
   - Syscalls: trap‑вход + таблица в BSD‑слое.
   - Минимальный userland: bootstrap‑порт/сервер.
   - Интеграция bootstrap с IDL/IPC.

2. Планировщик
   - MLQ/priority queues + таймслайс.
   - Политики приоритетов и базовая диагностика.

3. VFS
   - vnode/inode модель.
   - name cache + инвалидация.
   - mount + initrd/ramfs.

4. Учёт и права
   - real/effective UID/GID.
   - базовая модель прав.
   - MAC‑крючки (каркас).

5. Сеть
   - BSD‑сокеты (`AF_INET`), loopback.
   - UDP → TCP.

## Приоритет 1: Память

- Полный учет памяти по карте Multiboot2.
- PMM v2 с зонами и поддержкой дыр.
- Базовый VMM и kernel heap.

## Приоритет 2: Прерывания и таймеры

- Чистый IRQ путь без диагностического шума.
- Единая обработка EOI.
- Таймерный тик как основа планировщика.

## Приоритет 3: Планировщик

- Преэмптивное планирование.
- Минимальная очередь запуска.
- Корректный context switch.
- Приоритетный планировщик и таймслайс (MLQ/priority queues).

## Приоритет 4: VFS и RAMFS

- Интерфейсы VFS.
- RAM файловая система.
- Базовые shell-команды.
- Модель vnode/inode.
- Кэш имён (name cache).
- Монтирование и initrd/ramfs для старта.

## Приоритет 5: Пользовательский режим

- Syscall таблица.
- Переход CPL3.
- Минимальный загрузчик пользовательской программы.
- Bootstrap-порт и bootstrap-сервер (раздача портов сервисов).

## Приоритет 5.1: IPC

- Сообщения фиксированной и переменной длины.
- Возможность передачи портов в сообщениях.

## Приоритет 5.2: Syscalls и BSD-слой

- Таблица syscalls в BSD-слое.
- Вход через ловушку/переходник (traps) из userland.

## Приоритет 5.3: Учёт и права

- Реальные/эффективные UID/GID.
- Базовая модель прав.
- MAC‑крючки (позже).

## Приоритет 6.5: Сеть

- BSD‑сокеты (`AF_INET`).
- Порядок: loopback → UDP → TCP.

## Приоритет 6: Диагностика и тесты

- panic() с контекстом.
- Стабильный serial logger.
- Минимальные unit-тесты для ядра.

## Приоритет 7: IDL и генерация кода

- Ввести описание интерфейсов в `.defs`.
- Генератор кода для клиентских/серверных стабов и диспетчеров.
- Запрет ручного маршаллирования на межслойных границах.
